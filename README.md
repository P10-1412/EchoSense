## 介绍

**choSense** 是专为商业化与专业化发展的播客创作者打造的「静默运行内容价值雷达」，以AI驱动为核心，秉持"不替代创作、不打断表达"的隐形辅助理念，精准识别内容中的高商业价值、高传播潜力节点与高舆论风险点，提供全科视角的可落地建议，助力创作者更安全地表达、更聪明地变现、更高效地实现内容资产化。

### 核心定位

choSense 解决播客创作者的三大核心痛点：
- **创作决策难**：无法精准判断内容是否具备变现价值、出圈潜力
- **风险规避难**：难以预判舆论争议、掉粉等潜在风险
- **精力消耗大**：需投入大量时间进行内容分析、商业适配，分散创作注意力

### 四大差异化亮点

#### 1. 静默式体验
区别于传统"全量输出报告"的分析工具，choSense仅推送前1%极高价值和前10%高价值信息，极致降低干扰。

**静默运行原则**：
- 默认保持静默，不打扰创作流程
- 仅在检测到历史分位数≥90%的内容时主动提示
- 前1%极高价值内容自动弹窗重点提醒
- 其余时间界面简洁，让创作者专注创作

#### 2. 相对价值评估体系
不输出"绝对金额/粉丝量"，而是提供"相对价值+参考案例+可解释路径"，专业且可信。

**评估维度**：
- **历史分位数**：在历史数据中的排名（如"前5%"）
- **参考案例**：匹配相似场景的真实案例和数据
- **可解释路径**：清晰说明为什么这个内容有价值
- **采纳成本**：评估所需时间、执行难度、所需资源

**示例**：
```
相对价值等级：前5%
历史分位数：95%
参考案例：职场效率工具推荐场景
  - 受众规模：5-10万粉丝
  - 报价区间：8000-15000元
  - 数据来源：2024年职场类播客商业合作数据
采纳成本：
  - 所需时间：30分钟
  - 执行难度：易
  - 所需资源：口播脚本、产品链接
```

#### 3. 全科画像体系
基于法律、心理、商业、健康、传播五大核心学科，建立创作者长期画像，提供个性化、多维度支持。

**五大核心学科**：

**⚖️ 法律画像**
- 记录潜在违法表达、事实性风险、责任归属模糊表述
- 示例：绝对化广告宣传、不实信息传播、侵权风险

**🧠 心理画像**
- 记录信念模式、情绪叙事特点、潜在认知偏差
- 示例：完美主义倾向、过度焦虑、极端观念

**💼 商业/会计画像**
- 记录认知误区、低级错误、商业模式漏洞
- 示例：现金流概念混淆、投资风险忽视

**❤️ 健康/医学画像**
- 记录身体状态、生活习惯，提示潜在健康风险
- 注：不做疾病诊断，仅提示需专业检查的信号

**📡 传播学画像**
- 记录账号风格、受众特征、传播效果
- 用于辅助价值评估（不同账号发表相同观点的价值不同）

**动态主题学科**：
- 支持根据播客主题添加1-2个专业学科画像
- 示例：科技播客可添加"计算机科学"画像

#### 4. 可落地导向
所有建议均配套具体操作方案，而非空泛理论。

**商业化建议包含**：
- 商业适配度分析（自然嵌入可能性、受众明确度、观点闭环完整性）
- 适配广告形态（中插口播、品牌共创等）
- 口播脚本示例（可直接使用）
- 参考案例和报价区间

**传播价值建议包含**：
- 传播潜力分析（反直觉程度、冲突性、可切片性）
- 核心传播路径（小红书、知乎、抖音等）
- 内容策略（如何制作短视频切片）
- 预期效果数据

**风险预警建议包含**：
- 风险分析（极端度、不确定性、群体触及度）
- 潜在影响（如"评论区极化，互动率下降3-8%"）
- 原表述 vs 修改后表述（对比展示）
- 具体修改建议和缓冲话术

### 核心功能

#### 1. 智能内容分析
- **双模式输入**：支持播客URL自动提取或直接粘贴文字稿
- **静默运行**：默认不打扰，仅在关键节点出现
- **全科视角**：整合五大学科的专业分析

#### 2. 三类价值识别

**💰 商业化价值**
- 识别可商业化的情绪高峰点
- 评估广告植入、品牌合作的适配度
- 提供口播脚本和报价参考
- 计算采纳成本和预期收益

**🚀 传播价值**
- 识别可出圈的观点爆点
- 分析病毒传播潜力和社交媒体适配度
- 提供传播路径和内容策略
- 预估影响范围和互动效果

**⚠️ 风险预警**
- 检测潜在的舆论风险点
- 识别可能导致掉粉的敏感内容
- 提供风险规避建议和修改方案
- 参考历史争议案例

#### 3. 全科画像追踪
- 长期记录五大学科的发现和建议
- 可折叠展示，按需查看
- 支持动态添加主题学科
- 帮助创作者建立多维度认知

#### 4. 用户画像系统
- 记录账号风格和受众特征
- 考虑投资贴现率和风险承受能力
- 个性化调整价值评估标准
- 本地存储，保护隐私

### 使用流程

1. **输入内容**
   - 选择输入方式：播客URL或文字稿
   - 粘贴内容后点击"开始分析"

2. **静默分析**
   - 系统自动进行全科视角分析
   - 仅在检测到高价值/高风险内容时出现提示
   - 前1%极高价值内容自动弹窗

3. **查看建议**
   - 高价值节点识别（商业化、传播、风险）
   - 每条建议包含详细分析和可操作方案
   - 参考案例和相对价值评估

4. **查看画像**
   - 点击"查看全科画像"按钮
   - 浏览五大学科的历史记录
   - 了解长期认知模式和改进方向

5. **采纳建议**
   - 根据采纳成本评估优先级
   - 参考口播脚本、修改建议等具体方案
   - 实施后观察效果，持续优化

### 设计理念

- **深蓝色系主色调**（#1E3A8A），体现专业性和可信度
- **科技蓝辅助色**（#3B82F6），增强科技感和创新感
- **静默式UI**：默认简洁，仅在必要时显示信息
- **清晰的信息层次**：左侧高价值建议，右侧全科画像
- **响应式布局**：支持桌面和移动设备

## 本地开发

### 如何在本地编辑代码？

您可以选择 [VSCode](https://code.visualstudio.com/Download) 或者您常用的任何 IDE 编辑器，唯一的要求是安装 Node.js 和 npm.

### 环境要求

```
# Node.js ≥ 20
# npm ≥ 10
例如：
# node -v   # v20.18.3
# npm -v    # 10.8.2
```

具体安装步骤如下：

### 在 Windows 上安装 Node.js

```
# Step 1: 访问Node.js官网：https://nodejs.org/，点击下载后，会根据你的系统自动选择合适的版本（32位或64位）。
# Step 2: 运行安装程序：下载完成后，双击运行安装程序。
# Step 3: 完成安装：按照安装向导完成安装过程。
# Step 4: 验证安装：在命令提示符（cmd）或IDE终端（terminal）中输入 node -v 和 npm -v 来检查 Node.js 和 npm 是否正确安装。
```

### 在 macOS 上安装 Node.js

```
# Step 1: 使用Homebrew安装（推荐方法）：打开终端。输入命令brew install node并回车。如果尚未安装Homebrew，需要先安装Homebrew，
可以通过在终端中运行如下命令来安装：
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
或者使用官网安装程序：访问Node.js官网。下载macOS的.pkg安装包。打开下载的.pkg文件，按照提示完成安装。
# Step 2: 验证安装：在命令提示符（cmd）或IDE终端（terminal）中输入 node -v 和 npm -v 来检查 Node.js 和 npm 是否正确安装。
```

### 安装完后按照如下步骤操作：

```
# Step 1: 下载代码包
# Step 2: 解压代码包
# Step 3: 用IDE打开代码包，进入代码目录
# Step 4: IDE终端输入命令行，安装依赖：npm i
# Step 5: IDE终端输入命令行，启动开发服务器：npm run dev -- --host 127.0.0.1
```

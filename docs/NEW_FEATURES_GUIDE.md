# choSense 2.0 新功能使用指南

## 一、画像系统升级

### 核心改变
画像系统现在**仅记录客观事实**，不再包含建议。这使得画像更加客观、可信，便于长期追踪。

### 五大学科画像

#### 1. 心理画像 🧠
**记录内容**：
- 情绪模式（如"经常表现出焦虑情绪"）
- 认知特征（如"倾向于二元思维"）
- 信念系统（如"强调完美主义"）
- 压力反应模式（如"压力下容易情绪化"）

**示例**：
```
客观观察：
- 在讨论工作话题时，多次使用"必须"、"一定要"等绝对化表述
- 对失败的容忍度较低，表现出明显的完美主义倾向
- 情绪波动较大，从兴奋到沮丧转换迅速
```

#### 2. 传播学画像 📡
**记录内容**：
- 账号风格（如"知识分享型"、"娱乐搞笑型"）
- 受众特征（人口统计、兴趣偏好、互动模式）
- 内容主题和风格
- 更新频率和传播效果

**示例**：
```
客观观察：
- 受众主要为25-35岁职场人士
- 内容风格偏向深度分析，单期时长30-45分钟
- 平均互动率3.5%，高于行业平均水平
- 受众更喜欢实用技巧类内容，理论讨论互动较低
```

#### 3. 法律画像 ⚖️
**记录内容**：
- 法律风险领域
- 合规问题
- 免责声明使用情况

**示例**：
```
客观观察：
- 在推荐产品时未使用免责声明
- 对某行业做出绝对化评价，可能涉及商业诋毁风险
- 引用数据时未注明来源
```

#### 4. 商业画像 💼
**记录内容**：
- 商业认知水平
- 财务素养
- 变现历史

**示例**：
```
客观观察：
- 对现金流和利润的概念有混淆
- 讨论投资时未考虑风险分散
- 对商业模式的理解较为表面
```

#### 5. 健康画像 ❤️
**记录内容**：
- 生活方式模式
- 压力指标
- 健康意识水平

**示例**：
```
客观观察：
- 多次提及熬夜工作，睡眠时间不足
- 饮食不规律，经常跳过正餐
- 缺乏规律运动习惯
```

### 重复显示bug已修复
在DisciplineProfile组件中添加了去重逻辑，确保相同ID的记录不会重复显示。

## 二、历史记录系统

### 功能说明
完整记录每次分析的输入内容和生成的建议，方便回顾和对比。

### 使用方法

1. **查看历史记录**
   - 点击"历史记录"标签页
   - 查看所有历史分析记录

2. **筛选记录**
   - 点击"全部"、"商业化"、"传播"、"风险"按钮筛选
   - 仅显示包含对应类型建议的记录

3. **查看详情**
   - 点击任意记录的"查看详情"按钮
   - 查看完整的输入内容、建议列表、学科记录

### 记录内容
每条历史记录包含：
- 分析日期和时间
- 输入模式（URL或文字稿）
- 播客标题
- 生成的建议数量（按类型统计）
- 学科记录数量

## 三、详细设置系统

### 弹窗提醒设置

**触发阈值调整**：
- 范围：前1% ~ 前20%
- 前1%：仅极高价值内容弹窗（推荐）
- 前5%：很高价值内容弹窗
- 前10%：高价值内容弹窗
- 前20%：较高价值内容弹窗

**开关控制**：
- 可完全关闭弹窗提醒
- 关闭后仍会在页面中显示建议，只是不弹窗

### 分析内容筛选

**建议类型开关**：
- 💰 商业化价值分析：识别广告植入和商业合作机会
- 🚀 传播价值分析：识别可出圈的观点和话题
- ⚠️ 风险预警分析：检测潜在的舆论和法律风险

**使用场景**：
- 仅关注商业化：关闭传播和风险分析
- 仅关注风险：关闭商业化和传播分析
- 全面分析：保持所有开关开启（推荐）

### 学科画像设置

**五大学科开关**：
- ⚖️ 法律画像
- 🧠 心理画像
- 💼 商业画像
- ❤️ 健康画像
- 📡 传播学画像

**使用建议**：
- 根据个人关注点选择
- 建议至少保留心理和传播学画像
- 法律画像对风险规避很重要

### 分析详细程度

**三个级别**：
1. **基础分析**：快速识别关键点，适合快速浏览
2. **标准分析**：平衡速度和深度，推荐日常使用
3. **详细分析**：全面深入评估，适合重要内容

## 四、案例数据库系统

### 数据格式要求

**JSON格式示例**：
```json
{
  "id": "case_001",
  "type": "commercial",
  "date": "2024-12-19",
  "accountInfo": {
    "name": "职场成长笔记",
    "followers": "10-30万",
    "category": "职场发展",
    "style": "深度分析型"
  },
  "eventDescription": "在职场效率话题中植入时间管理工具广告",
  "eventResult": {
    "outcome": "positive",
    "revenue": 15000,
    "fansChange": 200,
    "engagementChange": 2.5,
    "details": "广告植入自然，受众反馈良好，转化率达到3.2%"
  },
  "tags": ["职场", "效率工具", "中插广告"],
  "source": "2024年Q4商业合作数据"
}
```

### 字段说明

**必填字段**：
- `id`: 唯一标识
- `type`: 案例类型（commercial/viral/risk）
- `date`: 发生时间
- `accountInfo`: 账号信息（name, followers, category, style）
- `eventDescription`: 事件描述
- `eventResult`: 事件结果（outcome, details）
- `tags`: 标签数组
- `source`: 数据来源

**可选字段**：
- `eventResult.revenue`: 收益（元）
- `eventResult.loss`: 损失（元）
- `eventResult.fansChange`: 粉丝变化
- `eventResult.engagementChange`: 互动率变化（%）

### 导入方法

**方式1：JSON导入**
1. 准备符合格式的JSON数据
2. 粘贴到导入文本框
3. 点击"导入JSON"按钮

**方式2：AI辅助整合**（UI已实现）
1. 粘贴非结构化文本（如案例描述）
2. 点击"AI辅助整合"按钮
3. AI自动提取关键信息并结构化
4. 检查并确认导入

### 导出方法
1. 点击"导出"按钮
2. 自动下载JSON文件
3. 文件名格式：`chosense-cases-{timestamp}.json`

### 数据库管理

**查看案例**：
- 按类型分类显示（商业化/传播/风险）
- 显示账号信息、事件描述、结果数据
- 标签云展示

**删除案例**：
- 点击单个案例的删除按钮
- 或点击"清空"按钮删除所有案例

## 五、数据库在分析中的应用

### 案例匹配逻辑

当进行价值分析时，系统会：
1. 从案例数据库中查找相似案例
2. 匹配标准：
   - 内容类别相似
   - 账号规模相近
   - 内容风格匹配
   - 标签重叠度高

3. 提供参考数据：
   - 相似场景的收益/损失范围
   - 粉丝和互动率变化
   - 成功/失败的关键因素

### 价值评估优化

**综合用户画像**：
- 根据传播学画像调整价值评估
- 考虑受众特征和账号风格
- 参考历史变现数据

**参考案例库**：
- 内置案例库（商业化4个、传播3个、风险3个）
- 用户导入的案例数据库
- 综合匹配，提供更精准的评估

## 七、常见问题

### Q1: 画像中为什么没有建议了？
A: 新版本画像仅记录客观事实，建议在"高价值节点识别"部分显示。这样设计是为了保持画像的客观性和长期价值。

### Q2: 如何查看历史分析的建议？
A: 进入"历史记录"标签页，点击任意记录的"查看详情"按钮。

### Q3: 案例数据库的数据会影响分析结果吗？
A: 会的。系统会从案例数据库中匹配相似案例，提供更精准的价值评估和参考数据。

### Q4: 可以导入CSV格式的案例数据吗？
A: 当前版本仅支持JSON格式。您可以使用"AI辅助整合"功能，粘贴非结构化文本让AI自动转换。

### Q5: 弹窗阈值设置为多少合适？
A: 推荐设置为前1%（默认值），这样只有极高价值的内容才会弹窗，避免打扰。如果希望看到更多建议，可以调整到前5%或前10%。

### Q6: 学科画像的重复显示bug修复了吗？
A: 是的，已在DisciplineProfile组件中添加了去重逻辑，相同ID的记录不会重复显示。

## 九、技术说明

### 数据存储
- 用户画像：localStorage (`chosense_user_profile`)
- 用户设置：localStorage (`chosense_settings`)
- 历史记录：localStorage (`chosense_history`)
- 案例数据库：localStorage (`chosense_cases`)

### 数据安全
- 所有数据存储在本地浏览器
- 不上传到服务器
- 清除浏览器数据会丢失所有记录
- 建议定期导出备份

### 组件位置
- 设置面板：`src/components/settings/SettingsPanel.tsx`
- 历史记录：`src/components/history/HistoryPanel.tsx`
- 案例数据库：`src/components/database/CaseDatabasePanel.tsx`
- 学科画像：`src/components/podcast/DisciplineProfile.tsx`
